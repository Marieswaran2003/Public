        QUESTION No.4


    ACME cooperation want to use the container technology, they want to create a custom container to run their application   

    Task:

    Build a custom image, utilize the initial file available at ~/projects/mariadb/acme-containerfile
    Base Image for the Container file mariadb:latest
    The container file should have build argument as below,

                    ACME_MARIADB_PASSWORD
                    ACME_MARIADB_DATABASE

    The container file should have environment variables as below,

                    MARIADB_ROOT_PASSWORD
                    MARIADB_DATABASE

    The environment variable should use build argument as below,

                    MARIADB_ROOT_PASSWORD use ACME_MARIADB_PASSWORD
                    MARIADB_DATABASE use ACME_MARIADB_DATABASE

    Use the Build argument should fetch it from user at build,

                    Variable name: ACME_MARIADB_PASSWORD
                    Variable value: acme
                    Variable name: ACME_MARIADB_DATABASE
                    Variable value: acme

    The Image must be locally available and tag as acme-mariadb:latest.
    Push the image to registry as acme:5000/acme-mariadb:latest

    amce-db-exporter:

    Build a custom image, utilize the initial file available at ~/projects/mariadb/acme_export_containerfile
    Base Image for the Container file mariadb:latest
    Copy the file from scripts/export.sh in to /scripts/export.sh in container
    Working directory is /scripts
    Run export.sh script
    The image must be locally available and tag as acme-mariadb-export:latest
    Push the image to registry as acme:5000/acme-mariadb-export:latest

    Testing your Work:

    a) Run a test container named acme-mariadb-test using the container image available at acme:5000/acme-mariadb:latest
    b) Test container should be in running state once initialized and persist to run.

    c) Run a test container named acme-mariadb-exporter using the container image available at acme:5000/acme-mariadb-export:latest    
        using the local map from path ~/workspace/acme-mariadb-db/export to /home in container.

    d) Once container execution is done, you should able to read the mysql dump file at the following
        location ~/workspace/acme-mariadb-db/export/acmeMysql.sql
    e) You can able to read the mysql dump with any reader.
